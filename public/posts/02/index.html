Clash for Linux

Posted at — Mar 1, 2020

开始

由于ssr在linux环境体验较差，于是开始使用clash。在此记录下过程及过程中遇到的坑。

环境

Linux debian 4.19.0-8-amd64

安装Clash-core

选择相应版本下载二进制文件并解压重命名为clash

#添加权限
chmod +x ./clash

若服务方没有提供配置文件可用下列命令生成

curl -f 你的托管地址 >> config.yml

详细配置文件见项目主页，按需配置。

clash默认配置文件目录为/home/你的用户名/.config/clash

我的配置文件目录有一个.yml文件和一个.yaml试了半天发现真正work的是.yaml

./clash

成功后如图

INFO[0000] SOCKS proxy listening at: 127.0.0.1:xxxx
INFO[0000] RESTful API listening at: 127.0.0.1:xxxx     
INFO[0000] HTTP proxy listening at: 127.0.0.1:xxxx

可用浏览器打开http://clash.razord.top/#/proxies进行管理

将clash添加到PATH

使用systemctl自动管理后台

新建clash管理文件

sudo vim /etc/systemd/system/clash.service

写入配置

[Unit]
Description=clash service
After=network.target

[Service]
User=zboy
ExecStart=/usr/local/bin/clash
Restart=on-abort

[Install]
WantedBy=multi-user.target

加载service

sudo systemctl daemon-reload

启动

service start clash.service

开机自启

systemctl enable clash.service

添加脚本自动更新配置文件

将预先个人配置存为custom.yml用于合成更新的配置

vim auto.sh

#添加权限
chmod +x auto.sh

贴上本人的垃圾脚本

#!/bin/zsh

curl -f 你的托管地址 >> config.yml #将clash托管地址转化为配置文件
echo "获取订阅成功"
cat custom.yml config.yml  > config.yaml #合并预先配置好的文件
mv -b -f config.yaml ~/.config/clash/ #移动到clash配置目录
rm config.yml
sudo service clash restart << EOF  #添加重定向自动输入sudo密码
你的sudo密码
EOF
echo "配置成功"

添加alias至.profile方便使用
